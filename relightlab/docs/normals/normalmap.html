Normal maps

A normal map encodes the surface orientation at every pixel as an (X, Y, Z) direction vector, remapped from the range [−1, 1] to the 8-bit range [0, 255].

ReLight can either compute a normal map from your image set using photometric stereo, or load an existing normal map image for further processing (flattening, surface integration).

Computation algorithm

L2 — Least Squares
Minimises the sum of squared residuals between the predicted and observed pixel intensities. Fast and robust when lighting coverage is good and there are no strong specular highlights or cast shadows. This is the recommended default for most datasets.
Robust
Uses an iteratively re-weighted scheme that down-weights outlier measurements, making it less sensitive to specular highlights, inter-reflections, and a small number of badly-lit images. Slower than L2 but produces cleaner normals on partially-shadowed surfaces.

Normal map format

Two conventions exist for how the Y (green) channel of a normal map is oriented. Loading a map in the wrong format will invert the apparent height of surface features (bumps appear as dents and vice-versa) and will produce incorrect surface integration.

OpenGL
The Y axis points upward in texture space (increasing V → up). This is the convention used by most DCC tools (Blender, Substance, Houdini) and by ReLight itself when it writes a computed normal map.
DirectX
The Y axis points downward in texture space (increasing V → down). This convention is used by Unreal Engine and some game-oriented export pipelines. When loading a DirectX normal map ReLight automatically flips the green channel before processing.

8-bit image vs. computed floating-point normals

When normals are computed from your image set, ReLight works internally with full 32-bit floating-point precision throughout the photometric stereo solve, flattening, and surface integration steps. The normal map is only quantised to 8 bits when it is written to disk.

When you load an existing normal map image (PNG or JPEG), the values are already quantised to 256 levels per channel. This introduces a small quantisation error in the direction vectors (≈ 0.4° at worst) which is generally invisible in shading but can produce visible staircase artefacts when the normals are integrated into a 3D surface — especially on gently curved areas where the height differences between adjacent pixels are very small.

For best surface quality, always integrate directly from computed normals rather than re-loading a saved normal map image.
